1. Resumen ejecutivo

Durante el periodo del 9 de marzo al 15 de abril de 2026 se ha diseñado, desarrollado y desplegado en producción la versión 1.0 del CRM Ofimatic Baix, un sistema de gestión comercial multi-empresa pensado para pymes españolas. El sistema cubre la totalidad del ciclo comercial: captación de leads, cualificación de prospectos, gestión de clientes activos, oportunidades de venta, actividades de seguimiento, visitas presenciales con geolocalización, importación masiva de datos heredados, y notificaciones nativas en móvil.

El proyecto se ha llevado a cabo aplicando una arquitectura SaaS moderna sobre infraestructura serverless (Vercel) y base de datos PostgreSQL gestionada (Supabase), con aislamiento estricto de datos entre empresas mediante seguridad a nivel de fila (RLS) y un sistema de planes de suscripción con límites configurables. Adicionalmente se ha empaquetado la aplicación como aplicación Android nativa mediante Capacitor, firmada y lista para distribución.

A lo largo del periodo se han registrado 140 entregas de código organizadas en trece fases funcionales, descritas en detalle a continuación. El producto resultante está actualmente en producción en crm-ofimaticbaix.vercel.app y es la herramienta de gestión comercial utilizada por Ofimatic Baix.

3. Fases del proyecto

Fase 1 — Fundación técnica y diseño visual (9 marzo)

Motivo: establecer una base sólida sobre la que iterar rápido durante las semanas siguientes, evitando deuda técnica desde el día uno.
  • Inicialización del proyecto en Next.js 16 con TypeScript estricto y Tailwind CSS.
  • Diseño del esquema de base de datos PostgreSQL completo (16+ tablas).
  • Sistema de tipos TypeScript generados a partir del esquema.
  • Castellanización completa de la interfaz (etapas, lifecycle, mensajes de error).
  • Diseño visual con efecto glassmorphism, soporte dark/light mode y branding inicial.

Fase 2 — Interfaz inicial y módulos visuales (10 marzo)

Motivo: validar la propuesta de valor visual antes de conectar a BD real, asegurando que el producto se sintiera moderno frente a Salesforce/HubSpot.
  • Rediseño completo del sidebar y dashboard (estilo minimalista tecnológico).
  • Página de configuración con persistencia en localStorage.
  • Optimización responsive móvil profesional (varias iteraciones).
  • Modales de creación: empresas, contactos, oportunidades, tareas.
  • Versión inicial de importación inteligente desde CSV/Excel.
  • Sub-páginas de clientes: activos, atrasados, cerrados.
  • KPIs en sidebar, página de métricas dedicada y mini-calendario interactivo.

Fase 3 — Backend real, autenticación y multi-tenancy (11–12 marzo)

Motivo: pasar de prototipo con datos simulados a sistema multi-empresa real con autenticación segura y aislamiento garantizado entre clientes.
  • Integración completa con Supabase Auth: registro, login, middleware de protección.
  • Páginas de login y signup con diseño SaaS profesional (glassmorphism + logo Ofimatic Baix).
  • Server Actions CRUD para contactos, empresas, oportunidades, pipeline y actividades.
  • Sistema multi-empresa (workspaces) con roles: propietario, administrador, miembro.
  • Onboarding automático: alta crea espacio, plan de prueba 14 días, pipeline + 7 etapas.
  • Panel super-admin para gestión global de suscripciones.
  • Correos privilegiados que ignoran restricciones de suscripción (Ofimatic Baix).

Fase 4 — Geolocalización, mapas y visitas comerciales (13 marzo)

Motivo: dotar al equipo comercial de Ofimatic Baix de herramientas en campo para planificación de rutas y registro de visitas.
  • Geocodificación automática vía OpenStreetMap/Nominatim con expansión de abreviaturas españolas (c/, ctra., pol., etc.) y cinco estrategias de reintento.
  • Tabla visits con check-in/out georreferenciados (lat, lon, dirección).
  • Tipos de visita: presencial, videollamada, llamada. Resultado: positivo, neutral, negativo, no show.
  • Duración calculada automáticamente en BD.
  • Tabla routes para planificación de rutas con múltiples empresas.
  • Página de mapa interactivo (Leaflet) con marcadores geocodificados.

Fase 5 — Webhooks e integraciones externas (13 marzo)

Motivo: permitir conectar el CRM con sistemas externos del cliente (n8n, ERPs, sistemas contables) sin polling manual.
  • Webhooks salientes configurables por empresa cliente.
  • Eventos: alta/modificación/baja de contactos, empresas, oportunidades y tareas; ganancia/pérdida/cambio de etapa; finalización de tarea.
  • Implementación robusta vía triggers PostgreSQL con pg_net: entrega garantizada aunque la web esté caída.
  • Logs auditables (webhook_logs) con código, cuerpo y duración; retención últimas 100 entradas por webhook.
  • Claves de API SHA-256 para consumidores externos.
  • API REST pública v1 (contactos, empresas, oportunidades).

Fase 6 — Rendimiento y experiencia de navegación (13 marzo)

Motivo: que la herramienta se percibiera tan rápida como Linear o Notion, no como un CRM tradicional.
  • Carga consolidada del contexto de espacio de trabajo en una sola consulta.
  • Caché cliente con invalidación selectiva en todas las páginas del dashboard.
  • Prefetching de rutas al pasar el cursor por la navegación.
  • Resultado: cambios de página percibidos < 100 ms tras la primera carga.

Fase 7 — Formularios completos, calendario y manual de usuario (13–14 marzo)

Motivo: llevar la captura de datos a nivel comercial y dotar al cliente final de documentación para autonomía.
  • Formularios completos de creación/edición de empresas y contactos.
  • Edición de tareas; corrección del bug de zona horaria del calendario.
  • "Días sin contacto" calculado con fecha real de finalización de actividad.
  • Clasificación automática mejorada (activo / atrasado / cerrado).
  • Manual de usuario completo en español para clientes finales.

Fase 8 — Importación inteligente avanzada (24 marzo)

Motivo: las pymes españolas llegan al CRM con datos en Excel, Access o exportaciones de ERPs antiguos. Sin importación robusta, no hay adopción.
  • Asistente de seis pasos: subida → detección → mapeo → limpieza → validación → inserción.
  • Formatos soportados: CSV, XLSX, XLS, MDB/ACCDB, JSON.
  • Detección inteligente de cabeceras en Excel con filas de título.
  • Auto-mapeo de columnas en tres capas: exacto, substring, patrón en valores.
  • Perfiles de importación reutilizables (guarda el mapeo).
  • Limpiadores por tipo: emails, teléfonos (corrige flotantes Excel), NIF/CIF, fechas (DD/MM/YYYY, seriales Excel), números formato europeo.
  • Tres estrategias de deduplicación: omitir, actualizar, importar igualmente.
  • Detección de duplicados por NIF, email o nombre con normalización Unicode.
  • Página dedicada de Clientes Potenciales.
  • Procesado por lotes de 50 con barra de progreso y descarga de errores en CSV.

Fase 9 — Unificación de Clientes y vinculación automática (24 marzo)

Motivo: los datos importados tenían contactos y empresas separados. Restablecer relaciones manualmente habría sido inviable.
  • Unificación de navegación: Contactos + Empresas agrupados bajo Clientes.
  • Algoritmo de vinculación automática con seis estrategias en cascada: nombre exacto, nombre parcial, email exacto, dominio de email, teléfono, coincidencia parcial.
  • API interna /api/internal/fix-contacts para diagnóstico y corrección masiva.
  • Recreación de contactos desde empresas para relaciones rotas.
  • Restauración de empresas eliminadas por error.

Fase 10 — Detalle de cliente y panel único (24 marzo)

Motivo: el flujo original obligaba a abrir varias páginas para ver un cliente; el cliente final solicitó vista 360° en un solo lugar.
  • Rediseño como panel único con todos los datos visibles.
  • Vinculación inline de contactos desde el propio panel.
  • Edición in situ sin modal aparte.
  • Campos solicitados expresamente: Población, Provincia, CIF, Código cliente, Forma de pago, Persona de contacto, Email 2-5, Teléfono secundario.
  • Botones de registro rápido (llamada, reunión, email, nota) con nota inline.
  • Confirmación visual "Registrado" tras cada acción.

Fase 11 — Vista general, navegación y experiencia de tablas (24 marzo)

Motivo: con cientos de clientes cargados, navegar y ordenar la información se volvió la prioridad de uso diario.
  • Vista General con fondo azul marino translúcido y buscador blanco contrastado.
  • Cabeceras de columna ordenables (asc / desc / sin orden) en todas las tablas.
  • Empresas ordenadas alfabéticamente por defecto.
  • Corrección de errores HTTP "Bad Request" en consultas grandes mediante división en lotes (máx. 200 por lote).
  • Click en fila completa abre el detalle; modal robusto frente a clicks neutros.
  • Sidebar reorganizado: Contactos y Empresas elevados al menú principal.
  • Límite de consulta elevado a 2.000 registros por defecto.

Fase 12 — Observabilidad y formularios de contacto (1–2 abril)

Motivo: preparar el sistema para producción real exige detección proactiva de errores y métricas reales.
  • Integración completa de Sentry (errores + rendimiento) con DSN configurado y rutas tunelizadas (/monitoring).
  • Integración de PostHog (instancia europea, conforme con RGPD) para analítica de producto.
  • Edición inline en panel de detalle de empresa (eliminación del modal aparte).
  • Crear contacto directamente desde el panel de empresa.
  • Rediseño del formulario de contacto con iconos, etiquetas y campo LinkedIn.
  • Unificación visual de formularios: notas, indicador de decisor, diseño coherente.
  • "Forma de pago" como desplegable con métodos B2B comunes (transferencia, domiciliación, pagaré, contado, confirming, etc.).

Fase 13 — Sobrevuelo pre-demo, Android y notificaciones (15 abril)

Motivo: preparar la demo comercial y entregar aplicación móvil real lista para distribución.
  • Pulido de UX general en todas las páginas del dashboard.
  • Drag-and-drop del pipeline con @dnd-kit, optimizado para escritorio (PointerSensor) y móvil (TouchSensor, umbral 6 px).
  • Acciones rápidas "Ganado / Perdido" en cada oportunidad.
  • Mejora del flujo de cambios de etapa del ciclo de vida del cliente.
  • Aplicación Android nativa con Capacitor 8, firmada con keystore propio (CRM-Ofimatic-v1.0.apk, app id com.ofimaticbaix.crm).
  • Sistema de notificaciones unificado: Web Notifications en navegador, @capacitor/local-notifications en APK.
  • Memoria anti-spam con limpieza automática a los 7 días.
  • Campanilla de notificaciones con buckets temporales (vencidas, < 1h, hoy, mañana, semana, después).
  • Banner superior de alertas de tareas urgentes con descarte por sesión.
  • Sonido de alerta generado por Web Audio API (sin archivos externos).
